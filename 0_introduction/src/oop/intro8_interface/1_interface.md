# Interface

**Интерфейс** в Java – это контракт, который определяет набор методов, которые класс должен реализовать, но не
предоставляет их реализации (за исключением методов с ключевыми словами ```default``` и ```static```, начиная с Java 8).
Интерфейсы позволяют отделить определение поведения от его реализации, что способствует гибкости и расширяемости кода.

## Как создать интерфейс

Интерфейс объявляется с использованием ключевого слова **interface**. Все методы, объявленные в интерфейсе, по умолчанию
являются ```public``` и ```abstract``` (если не заданы как ```default``` или ```static```). Интерфейсы могут также
содержать константы (```static final```-поля).

```java
public interface Animal {
    // Абстрактный метод – реализация будет в классах, которые реализуют этот интерфейс
    void makeSound();

    // Default-метод с реализацией, начиная с Java 8
    default void sleep() {
        System.out.println("Animal is sleeping");
    }

    // Статический метод, также начиная с Java 8
    static void describe() {
        System.out.println("This is an Animal interface");
    }
}

public class Dog implements Animal {
    @Override
    public void makeSound() {
        System.out.println("Гав-гав");
    }

    // Метод sleep() можно не переопределять, так как он уже имеет реализацию в интерфейсе
}
```

## Для чего необходимы интерфейсы

- **Определять контракты**. Задавая набор методов без реализации, интерфейсы гарантируют, что все классы, их
  реализующие, будут обладать одним и тем же поведением. Это облегчает заменяемость и расширяемость кода.
- **Множественное наследование типа**. Класс в Java может наследовать только один суперкласс, но может реализовывать
  множество интерфейсов. Это позволяет комбинировать разные аспекты поведения.
- **Слабую связанность**. Использование интерфейсов способствует созданию слабо связанных систем, что упрощает модульное
  тестирование и поддержку кода.

## Разница между интерфейсом и абстрактным классом

| Характеристика         | Интерфейс                                                                                | Абстрактный класс                                                              |
|------------------------|------------------------------------------------------------------------------------------|--------------------------------------------------------------------------------|
| **Наследование**       | Класс может реализовывать множество интерфейсов                                          | Класс может наследоваться только от одного абстрактного класса                 |
| **Реализация методов** | До Java 8 все методы были абстрактными; с Java 8 – default и static методы с реализацией | Может содержать как абстрактные методы, так и методы с реализацией             |
| **Поля**               | Может содержать только static final (константы)                                          | Может содержать любые поля (instance variables), позволяющие хранить состояние |
| **Конструкторы**       | Не могут иметь конструкторы                                                              | Могут иметь конструкторы для инициализации полей                               |
| **Цель использования** | Определение набора обязанностей, которые должны выполнять классы                         | Создание общей базовой реализации для группы классов                           |

## "Мнемоническое" правило

_ДАННЫЕ ПРАВИЛА СОБЛЮДАЮТСЯ НЕ ВСЕГДА_

- Зачастую класс называется Существительным, а интерфейсы довольно часто имеют названия в виде глагола или
  прилагательного.
- **Наследование классов.** Когда класс наследует класс это можно описать отношениям _является_. Если ```B```
  наследуется от ```A```, то можно утвержадть что ```B``` также является ```А```. Класс ```Person``` наследуется от
  класса ```Animal``` - Человек является Животным. Класс ```Student``` наследует ```Person``` - Студент является
  человеком.
- **Реализация интерфейсов.** Когда класс реализуется интерфейс, то он преобритает поведение, описанное в интерфейсе.
  Например если класс реализуется интерфейс ```Serializable``` - означает что класс может быть сериализован, то есть он
  приобритает такое поведение как возможность сериализации.

**Ну а вообще выбор между классом, абстрактном классом и интерфейсом зависит от ситуации и требований.**
