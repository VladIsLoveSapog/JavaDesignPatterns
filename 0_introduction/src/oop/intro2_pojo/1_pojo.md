# POJO

**POJO** (Plain Old Java Object) – это простой объект Java, который не зависит от специфических фреймворков, не
наследует никаких специальных классов и не требует соблюдения жёстких контрактов (как, например, у EJB). По сути, POJO –
это обычный Java-объект, предназначенный для хранения данных и/или реализации бизнес-логики без лишней "магии". Он
характеризуется простотой, минимальными ограничениями и высокой читаемостью.

## Основные характеристики POJO

- **Минимальные зависимости:** POJO не привязан к каким-либо специфическим фреймворкам или библиотекам. Его можно
  использовать в самых разных приложениях, не опасаясь, что придётся соблюдать жесткие правила или привязываться к
  конкретной инфраструктуре.
- **Отсутствие специальных аннотаций или наследования:** POJO не требует, чтобы класс расширял какой-либо базовый класс
  или реализовывал специальные интерфейсы (если, конечно, это не нужно по логике приложения). Это делает такие объекты
  легковесными и удобными для тестирования.
- **Часто используются для представления данных:** Обычно POJO-классы имеют приватные поля и публичные методы доступа (
  геттеры и сеттеры), хотя это не является строгим требованием. Такая структура позволяет инкапсулировать данные и
  обеспечивает контроль над их изменением.

### Пример

Ниже приведён пример простого POJO-класса, представляющего сущность «Человек»:

```java
public class Person {
    // Приватные поля для хранения данных
    private String name;
    private int age;

    // Конструктор по умолчанию
    public Person() {
    }

    // Конструктор с параметрами для удобства создания объекта с начальными значениями
    public Person(String name, int age) {
        this.name = name;
        this.age = age;
    }

    // Геттер для получения имени
    public String getName() {
        return name;
    }

    // Сеттер для установки имени
    public void setName(String name) {
        this.name = name;
    }

    // Геттер для получения возраста
    public int getAge() {
        return age;
    }

    // Сеттер для установки возраста
    public void setAge(int age) {
        this.age = age;
    }
}
```

В этом примере класс ```Person``` является POJO, поскольку:

- Он не наследует никаких специальных классов и не реализует специфических интерфейсов.
- Он использует простые приватные поля для хранения данных.
- Предоставляет публичные геттеры и сеттеры для доступа к этим данным.
- Его конструкция не обременена дополнительной логикой, не связанной с основной задачей представления данных.
