# Наследование

**Наследование** – это один из ключевых механизмов объектно-ориентированного программирования в Java, позволяющий
создавать новый класс на основе уже существующего. Новый класс (называемый подклассом или наследником) получает все
доступные (не private) поля и методы родительского класса (суперкласса), что позволяет повторно использовать код и
расширять функциональность без его дублирования.

## Ключевое слово для наследования

В Java для указания наследования используется ключевое слово ```extends```. Если вы создаёте класс-наследник, вы пишете
его следующим образом:

```java
public class Parent {
    // поля и методы суперкласса
}

public class Child extends Parent {
    // дополнительные поля и методы, а также возможность переопределять методы суперкласса
}
```

## Ограничения наследования в Java

### Одиночное наследование классов

В Java класс может наследовать только один прямой родительский класс. Это означает, что множественное наследование (
наследование от более чем одного класса) не поддерживается. Однако, для реализации функциональности от нескольких
источников можно использовать **интерфейсы** (с ключевым словом ```implements```), так как класс может реализовывать
несколько интерфейсов.

### Наследование final-классов

Класс, объявленный с модификатором ```final```, не может быть унаследован. Это сделано для обеспечения безопасности и
неизменности логики класса. Например, класс ```String``` объявлен как ```final```, поэтому его нельзя расширять.

```java
public final class ImmutableClass {
    // код класса
}

// Ошибка: нельзя наследовать final класс
public class ExtendedClass extends ImmutableClass {
}
```

### Наследование final-методов

Если метод суперкласса объявлен как ```final```, его нельзя переопределить в подклассе. Это гарантирует, что критически
важная логика останется неизменной.

```java
public class Parent {
    public final void display() {
        System.out.println("Неизменяемый метод");
    }
}

public class Child extends Parent {
    // Ошибка: нельзя переопределить метод display()
    // public void display() { ... }
}
```

### Доступ к членам суперкласса

Члены суперкласса, объявленные с модификатором **private**, недоступны в подклассе напрямую. Чтобы обеспечить доступ к
таким данным, часто используют **геттеры/сеттеры** или **protected**-модификатор, который позволяет наследникам получать
доступ к членам суперкласса.
