# ООП

**ООП** – это парадигма программирования, которая основывается на использовании объектов. Основная идея ООП заключается
в том, чтобы моделировать реальные сущности как объекты, обладающие состоянием и поведением. Это помогает
структурировать программу так, чтобы код был более модульным, читаемым и поддерживаемым.

## Основные принципы ООП

- Инкапсуляция
- Наследование
- Полиморфизм

_Еще иногда в разных источниках добавляют **Абстракцию**_

## Абстракция

**Абстракция** как принцип ООП, в моем понимании это не про абстрактные классы, а про то что в классах и в следствии в
объектах мы описываим лишь необходимую в нашем случае информацию о какой-то сущности.

**_Пример:_** автомобиль, тот который можно встретить на улице, является большой, сложной, комплексной системой. Но
допустим в нашей программе небоходимы лишь следующие сведения б автомобиле:

- кол-во мест
- объем двигателя
- гос. номер

Таким образом написав класс следеющего вида:

```java
class Auto {
    private Integer seatNumber;
    private Integer engineVolume;
    private String number;

    // и тут ещё конструкторы, геттеры, сеттеры и тд.
}
```

Мы создали свою абстракцию машины в нашей программе, которая содержит лишь то что нам необходимо.

## Инкапсуляция

**Инкапсуляция** – это один из основных принципов объектно-ориентированного программирования (ООП), который позволяет
объединить данные (поля) и методы, работающие с этими данными, в одном классе, скрывая внутреннюю реализацию от внешнего
мира. Благодаря этому достигается защита данных и упрощается сопровождение кода, так как любые изменения внутри класса
не затрагивают его пользователей, если интерфейс остаётся неизменным.

### Основные аспекты инкапсуляции

- **Сокрытие данных:** Поля класса объявляются с минимально необходимым уровнем доступа, чаще всего как ```private```,
  чтобы предотвратить прямое изменение из внешнего кода.
- **Доступ через методы:** Взаимодействие с данными происходит через публичные методы (например, геттеры и сеттеры) или
  специализированные методы, которые могут включать валидацию и другие проверки.
- **Контроль изменений:** Класс сам контролирует, как и когда его данные могут изменяться, что помогает избежать ошибок
  и нарушений логики работы программы.

### Модификаторы доступа в Java

Модификаторы доступа позволяют задавать уровни видимости для классов, полей и методов. В Java используются следующие
модификаторы:

1. **private** - доступен только внутри того же класса.
2. **default (package-private)** - Если не указан модификатор доступа, элемент доступен внутри всего пакета, в котором
   он объявлен. **Пример:** если в классе не указать модификатор для метода или поля, то этот метод/поле будет доступно
   всем классам внутри одного пакета.
3. **protected** - доступен внутри класс и в подклассах.
4. **public** - доступен отовсюду – из любого класса и пакета.

## Наследование

**Наследование** – это один из основных принципов объектно-ориентированного программирования, позволяющий создавать
новый класс на основе уже существующего. Новый класс (называемый подклассом или наследником) автоматически получает поля
и методы родительского класса (суперкласса), что способствует повторному использованию кода и упрощает его поддержку.

### Наследование позволяет

- **Повторно использовать код**. Общая функциональность, реализованная в родительском классе, доступна и в его
  подклассах.
- **Расширять функциональность**. Подклассы могут добавлять новые поля и методы или переопределять (```override```)
  методы суперкласса, чтобы изменить или дополнить поведение.
- **Упрощать архитектуру программы.** За счет иерархии классов становится проще моделировать реальные объекты, где более
  конкретные классы наследуют общие черты от более абстрактных.

### Пример

Рассмотрим простой пример, где у нас есть базовый класс ```Animal``` (Животное), от которого наследуются конкретные
классы ```Dog``` (Собака) и ```Cat``` (Кошка).

```java
// Родительский класс
public class Animal {
    // Защищённое поле, доступное в классе и его наследниках
    protected String name;

    public Animal(String name) {
        this.name = name;
    }

    // Метод, который может быть переопределён в наследниках
    public void makeSound() {
        System.out.println("Некоторый звук");
    }
}

// Подкласс Dog, наследующий класс Animal
public class Dog extends Animal {

    public Dog(String name) {
        super(name); // Вызов конструктора родительского класса
    }

    // Переопределяем метод makeSound для конкретного поведения собаки
    @Override
    public void makeSound() {
        System.out.println("Гав-гав");
    }
}

// Подкласс Cat, наследующий класс Animal
public class Cat extends Animal {

    public Cat(String name) {
        super(name);
    }

    // Переопределяем метод makeSound для кошки
    @Override
    public void makeSound() {
        System.out.println("Мяу");
    }
}

// Пример использования наследования в методе main
public class Main {
    public static void main(String[] args) {
        Animal animal1 = new Dog("Бобик");
        Animal animal2 = new Cat("Мурка");

        animal1.makeSound(); // Выведет: Гав-гав
        animal2.makeSound(); // Выведет: Мяу
    }
}
```

## Полиморфизм

**Полиморфизм** – один из ключевых принципов ООП, позволяющий объектам вести себя по-разному в зависимости от их
конкретного типа, даже если они объявлены как объекты более общего типа (суперкласса или интерфейса). Это даёт
возможность писать более гибкий и расширяемый код.

### Историческая справка откуда взялось это слово

Слово «полиморфизм» происходит от греческих «поли» – много и «морф» – форма. В контексте ООП это означает, что один и
тот же метод может иметь несколько реализаций, и вызов этого метода зависит от того, к какому конкретному классу
принадлежит объект в момент выполнения программы.

### Виды полиморфизма

- **Динамический полиморфизм (переопределение методов)** Он достигается за счёт того, что подклассы переопределяют
  методы суперкласса. При вызове метода через переменную типа суперкласса будет выполнена версия, соответствующая
  реальному типу объекта, а не типу ссылки.
- **Статический полиморфизм (перегрузка методов)** Здесь несколько методов имеют одинаковое имя, но различаются списком
  параметров. Решение о том, какую версию метода вызвать, принимается на этапе компиляции.

### Пример динамического полиморфизма

Рассмотрим классическую ситуацию с базовым классом ```Animal``` и его наследниками ```Dog``` и ```Cat```.

```java
// Родительский класс
public class Animal {
    // Публичный метод, который будем переопределять
    public void makeSound() {
        System.out.println("Некоторый звук");
    }
}

// Подкласс Dog переопределяет метод makeSound()
public class Dog extends Animal {
    @Override
    public void makeSound() {
        System.out.println("Гав-гав");
    }
}

// Подкласс Cat переопределяет метод makeSound()
public class Cat extends Animal {
    @Override
    public void makeSound() {
        System.out.println("Мяу");
    }
}

// Использование полиморфизма в методе main
public class Main {
    public static void main(String[] args) {
        // Переменные типа Animal могут ссылаться на объекты Dog и Cat
        Animal animal1 = new Dog();
        Animal animal2 = new Cat();

        // Вызов метода makeSound() определяется динамически – по реальному типу объекта
        animal1.makeSound(); // Выведет: Гав-гав
        animal2.makeSound(); // Выведет: Мяу
    }
}
```

В данном примере мы видим, что несмотря на то, что переменные ```animal1``` и ```animal2``` имеют тип ```Animal```, при
вызове метода ```makeSound()``` срабатывает та реализация, которая соответствует фактическому типу объекта (```Dog```
или ```Cat```). Это и есть динамический полиморфизм.

### Пример статического полиморфизма (перегрузка методов)

Статический полиморфизм достигается за счёт перегрузки методов. Здесь решение о том, какой метод вызывать, принимается
на этапе компиляции.

```java
public class Calculator {
    // Метод для сложения целых чисел
    public int add(int a, int b) {
        return a + b;
    }

    // Перегруженный метод для сложения чисел с плавающей точкой
    public double add(double a, double b) {
        return a + b;
    }

    // Перегруженный метод для сложения трёх чисел
    public int add(int a, int b, int c) {
        return a + b + c;
    }

    public static void main(String[] args) {
        Calculator calc = new Calculator();
        System.out.println(calc.add(2, 3));        // Вызов первого метода, выведет 5
        System.out.println(calc.add(2.5, 3.5));      // Вызов второго метода, выведет 6.0
        System.out.println(calc.add(1, 2, 3));       // Вызов третьего метода, выведет 6
    }
}
```

Здесь метод ```add``` существует в нескольких вариантах, и компилятор определяет нужный вариант по типам и количеству
параметров.
