# Bean

**Bean** объект, чаще всего называемый **JavaBean**, – это специальный тип Java-объекта, который следует определённым
конвенциям для упрощения его использования, повторного применения и интеграции с различными фреймворками. Он
представляет собой обычный класс, но с некоторыми дополнительными требованиями, позволяющими инструментам и библиотекам
автоматически управлять его свойствами.

## Основные требования JavaBean

1. **Публичный конструктор по умолчанию:** У класса должен быть конструктор без параметров, чтобы его можно было легко
   создавать через механизмы рефлексии или при десериализации.
2. **Приватные поля (свойства):** Данные класса обычно объявляются как private. Это способствует сокрытию данных (
   инкапсуляции) и защищает их от некорректного изменения.
3. **Публичные методы доступа (геттеры и сеттеры):** Для каждого приватного поля следует предоставить методы для чтения
   и записи значений. При этом имена методов должны соответствовать соглашениям: для свойства ```property``` геттер
   называется ```getProperty()```, а сеттер – ```setProperty(...)```. Для булевых значений может использоваться
   префикс ```is``` (например, ```isActive()```).
4. **Реализация Serializable (опционально)**:
   Часто JavaBeans реализуют интерфейс ```Serializable``` для возможности сохранения их состояния (например, для
   передачи по сети или записи в файл).

## Пример JavaBean

Ниже приведён пример простого JavaBean, описывающего сущность «Человек»:

```java
import java.io.Serializable;

public class PersonBean implements Serializable {
    // Приватные поля для хранения данных
    private String name;
    private int age;

    // Публичный конструктор по умолчанию
    public PersonBean() {
    }

    // Конструктор с параметрами (не является обязательным, но может быть полезен)
    public PersonBean(String name, int age) {
        this.name = name;
        this.age = age;
    }

    // Геттер для поля name
    public String getName() {
        return name;
    }

    // Сеттер для поля name
    public void setName(String name) {
        this.name = name;
    }

    // Геттер для поля age
    public int getAge() {
        return age;
    }

    // Сеттер для поля age
    public void setAge(int age) {
        this.age = age;
    }
}
```

## Зачем нужны JavaBean

- **Унификация и стандартизация.** Следуя установленным конвенциям, JavaBeans можно легко использовать в различных
  инструментах разработки, например, в визуальных редакторах пользовательских интерфейсов или в ORM-фреймворках (
  Hibernate, MyBatis и т.д.).
- **Интеграция с фреймворками.** Многие современные фреймворки (например, Spring) используют термин «bean» для
  обозначения объектов, управляемых контейнером. Такой контейнер отвечает за создание, настройку и связывание объектов
  между собой (dependency injection). Несмотря на то, что в Spring bean может не полностью соответствовать классическим
  требованиям JavaBean, основная идея – иметь управляемый, настраиваемый компонент – остаётся той же.
