# Abstract

Ключевое слово **abstract** используется для создания абстрактных классов и методов, которые служат шаблонами для
будущих конкретных реализаций. Они позволяют задать общий контракт для группы классов, не предоставляя полной
реализации, а оставляя детали для конкретных наследников.

## Абстрактные классы

**Абстрактный класс** – это класс, который объявлен с ключевым словом ```abstract```. Такой класс не может быть
инстанциирован напрямую (то есть, нельзя создать объект этого класса через ```new```). Он служит основой для создания
подклассов, которые будут предоставлять конкретную реализацию абстрактных методов.

**Особенности абстрактного класса**

- Может содержать как абстрактные методы (без реализации), так и обычные (конкретные) методы с реализацией.
- Может содержать поля и конструкторы.
- Предназначен для объединения общего функционала, который разделяют все наследники.

## Абстрактные методы

Абстрактный метод объявляется без тела (без фигурных скобок) и заканчивается точкой с запятой. Он определяет только
сигнатуру метода, оставляя реализацию для классов-наследников.

**Особенности абстрактного метода**

- Обязательно должен быть реализован в первом конкретном (не абстрактном) классе-наследнике.
- Абстрактный метод может быть объявлен только внутри абстрактного класса.

## Пример

```java
// Абстрактный класс Animal, который задает общий контракт для всех животных
public abstract class Animal {
    private String name;

    // Конструктор для инициализации поля name
    public Animal(String name) {
        this.name = name;
    }

    // Абстрактный метод, который не имеет реализации.
    // Конкретные животные должны реализовать этот метод.
    public abstract void makeSound();

    // Конкретный метод, который может использоваться всеми наследниками
    public void displayInfo() {
        System.out.println("Животное: " + name);
    }
}

// Класс Dog наследует Animal и предоставляет реализацию абстрактного метода makeSound()
public class Dog extends Animal {
    public Dog(String name) {
        super(name);
    }

    // Переопределение абстрактного метода makeSound()
    @Override
    public void makeSound() {
        System.out.println("Гав-гав");
    }
}

// Пример использования
public class Main {
    public static void main(String[] args) {
        // Нельзя создать объект абстрактного класса Animal:
        // Animal animal = new Animal("Лев"); // Ошибка компиляции!

        // Создаем объект класса Dog, который является конкретной реализацией Animal
        Animal dog = new Dog("Бобик");
        dog.displayInfo();   // Выведет: Животное: Бобик
        dog.makeSound();     // Выведет: Гав-гав
    }
}
```

- ```Animal``` – абстрактный класс, содержащий поле name, конструктор, один абстрактный метод ```makeSound()``` и один
  конкретный метод ```displayInfo()```.
- ```Dog``` – конкретный класс-наследник, который обязан реализовать метод ```makeSound()```. Благодаря этому, каждый
  конкретный класс, представляющий определённое животное, может реализовывать звук по-своему.
- В методе ```main``` демонстрируется, что нельзя создать экземпляр абстрактного класса ```Animal```, но можно создать
  объект класса ```Dog```, который является конкретной реализацией.
