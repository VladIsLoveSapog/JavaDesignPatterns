# Основной поток выполнения программы

В Java-приложениях **главный поток** (`main` thread) — это начальный поток выполнения, который создаётся виртуальной машиной Java (JVM) при запуске программы. Он отвечает за выполнение метода `main()` и служит точкой входа в приложение.

```java
public class Main {
    public static void main(String[] args) {
        // код....
    }
}
```

---

## Запуск JVM и создание главного потока

Пусть `MyApp` некий класс с функцией `main` нашего приложения.

Когда пользователь запускает `java MyApp`, происходит следующая последовательность:

1. **ОС создаёт процесс JVM** — выделяется адресное пространство, загружается `java.exe` / `libjvm.so`.
2. **Инициализация JVM** — выделяется куча (heap), стеки (stack), настраивается сборщик мусора (GC), загружаются bootstrap-классы (`rt.jar` / `java.base`).
3. **Создание главного потока** — JVM создаёт объект `Thread` со следующими параметрами:
   - `getName()` → `"main"`
   - `getThreadGroup().getName()` → `"main"`
   - `getPriority()` → `5` (`Thread.NORM_PRIORITY`)
   - `isDaemon()` → `false`
4. **Загрузка входного класса** — JVM находит класс `MyApp`, выполняет загрузку, верификацию байткода и статическую инициализацию (`static {}`-блоки).
5. **Вызов `main(String[] args)`** — JVM вызывает метод `main()` в контексте главного потока.
6. **Завершение** — когда `main()` возвращает управление:
   - если не-демонических (non-daemon) потоков больше нет → JVM завершает работу;
   - если есть активные не-демонические потоки → JVM ожидает их завершения.

```
java MyApp
    │
    ▼
ОС: создаёт процесс JVM
    │
    ▼
JVM: инициализация (heap, GC, bootstrap-классы)
    │
    ▼
JVM: создаёт главный поток (name="main", priority=5, daemon=false)
    │
    ▼
JVM: загружает MyApp → статическая инициализация
    │
    ▼
Главный поток: вызывает MyApp.main(args)
    │
    ▼
main() завершился → нет non-daemon потоков? → JVM exit
                  → есть non-daemon потоки?  → JVM ждёт
```

## Жизненный цикл главного потока

Главный поток проходит те же состояния, что и любой другой поток:

| Состояние        | Описание                                                                 |
|------------------|--------------------------------------------------------------------------|
| `NEW`            | Поток создан, но ещё не запущен (для главного потока это мгновенный этап) |
| `RUNNABLE`       | Поток выполняется или готов к выполнению — ожидает процессорного времени  |
| `BLOCKED`        | Поток ожидает монитора (вход в `synchronized`-блок)                       |
| `WAITING`        | Поток ожидает бесконечно: `Object.wait()`, `Thread.join()`               |
| `TIMED_WAITING`  | Поток ожидает ограниченное время: `Thread.sleep()`, `Object.wait(long)`  |
| `TERMINATED`     | Выполнение метода `run()` завершено                                       |

**Необработанное исключение в `main()`** — если из `main()` вылетает непойманное исключение, JVM вызывает зарегистрированный `UncaughtExceptionHandler` (по умолчанию печатает стектрейс в `System.err`), после чего главный поток переходит в `TERMINATED`.

**Демонические потоки** — потоки, помеченные как `daemon`, не препятствуют завершению JVM. Если главный поток завершился, а остались только демонические потоки, JVM немедленно останавливается, принудительно прерывая все демонические потоки.

## Характеристики главного потока

- **Имя потока**: по умолчанию `"main"`.
- **Группа потоков**: группа `"main"`, если не указано иное.
- **Приоритет**: `Thread.NORM_PRIORITY` = `5`.
- **Демон**: `isDaemon()` возвращает `false` — главный поток никогда не является демоном.
- **Жизненный цикл**: существует до завершения `main()` и всех порождённых не-демонических потоков.

## Роль главного потока

- **Точка входа**: главный поток служит начальной точкой выполнения программы.
- **Создание новых потоков**: из главного потока запускаются дочерние потоки для параллельных задач.
- **Завершение программы**: программа продолжает работать, пока живут все не-демонические потоки, включая главный.
