# Deadlock

**Deadlock** — это состояние, при котором два или более потоков навсегда блокируют друг друга, ожидая освобождения
ресурсов, которые находятся под контролем других потоков. В результате ни один из потоков не может продолжить
выполнение, что приводит к зависанию приложения.

## Зачем важно понимать Deadlock

- **Сложность обнаружения**: Deadlock не вызывает исключений и не выводит ошибок — программа просто «замирает». Без понимания механизма его возникновения отладка может занять часы.
- **Воспроизводимость**: Deadlock зависит от порядка планирования потоков, поэтому может не воспроизводиться при повторном запуске, но стабильно возникать в продакшене под нагрузкой.
- **Каскадный эффект**: Заблокированные потоки удерживают ресурсы, которые нужны другим частям системы, что может привести к полной остановке приложения.

## Причины возникновения Deadlock

Deadlock возникает, когда одновременно выполняются четыре условия, известные как **условия Coffman**:

1. **Взаимное исключение (Mutual Exclusion)**: Ресурс может быть захвачен только одним потоком за раз.
2. **Удержание и ожидание (Hold and Wait)**: Поток удерживает по крайней мере один ресурс и ожидает захвата
   дополнительных ресурсов, удерживаемых другими потоками.
3. **Отсутствие принудительного освобождения (No Preemption)**: Ресурсы не могут быть принудительно освобождены; они
   могут быть освобождены только потоком, который их захватил.
4. **Циклическое ожидание (Circular Wait)**: Существует цикл из двух или более потоков, каждый из которых ожидает
   ресурса, удерживаемого следующим потоком в цикле.

## Пример Deadlock-а

[DeadlockExample.java](code/DeadlockExample.java) — два потока захватывают два ресурса в **разном** порядке, что приводит к взаимной блокировке.

1. **Поток-1** захватывает `Ресурс A` и затем пытается захватить `Ресурс B`.
2. **Поток-2** захватывает `Ресурс B` и затем пытается захватить `Ресурс A`.
3. Оба потока оказываются в состоянии ожидания друг друга, создавая **Deadlock**.

> При запуске программа **зависнет** — это ожидаемое поведение.

## Как предотвратить Deadlock

Достаточно нарушить **хотя бы одно** из условий Coffman:

1. **Избегать циклических зависимостей (упорядочивание ресурсов)**: Все потоки захватывают ресурсы в одном и том же порядке. Это самый надёжный и распространённый способ.
2. **Использовать тайм-ауты**: Попытка захватить ресурс с тайм-аутом (`tryLock` с `timeout`), чтобы избежать бесконечного ожидания.
3. **Минимизировать количество захватываемых ресурсов**: Ограничить число ресурсов, которые каждый поток захватывает одновременно.
4. **Использовать высокоуровневые механизмы**: Классы из пакета `java.util.concurrent` (например, `ReentrantLock` с `tryLock`), которые предоставляют более гибкое управление блокировками.

## Пример предотвращения Deadlock

[DeadlockFixExample.java](code/DeadlockFixExample.java) — те же два потока и два ресурса, но оба потока захватывают ресурсы **в одном порядке** (A → B). Циклическое ожидание невозможно, программа завершается корректно.
