# Спящий парикмахер (Sleeping Barber)

## Описание задачи

В парикмахерской работает один парикмахер. Когда клиентов нет — он спит. В зале ожидания есть ограниченное число мест. Приходящий клиент либо будит парикмахера (если тот спит), либо садится в зал ожидания (если парикмахер занят), либо уходит (если мест нет).

## Какие проблемы иллюстрирует задача

### Производитель-Потребитель (Producer-Consumer)

Клиенты = производители запросов на стрижку, парикмахер = потребитель, обрабатывающий запросы по одному. Классическая схема, где один поток генерирует работу, а другой её выполняет.

### Ограниченный буфер (Bounded Buffer)

Зал ожидания — это очередь фиксированного размера. Если буфер полон (все места заняты), новый элемент (клиент) отбрасывается. Это модель ограниченной очереди задач.

### Координация wait/notify

Парикмахер должен спать, когда клиентов нет, и просыпаться по сигналу. Это прямое применение `wait()` (парикмахер засыпает) и `notify()` (клиент будит).

## Примеры

### [BarberWaitNotify.java](code/BarberWaitNotify.java) — решение через synchronized + wait/notify

Зал ожидания реализован как `synchronized`-объект с очередью. Парикмахер вызывает `wait()`, когда очередь пуста; клиент вызывает `notifyAll()`, когда садится. Метод `closeShop()` позволяет корректно завершить программу.

### [BarberSemaphore.java](code/BarberSemaphore.java) — решение через Semaphore

Три семафора координируют работу: `waitingRoom` ограничивает места, `customerReady` будит парикмахера, `barberReady` сигнализирует клиенту. Неблокирующий `tryAcquire()` позволяет клиенту уйти сразу, если мест нет.

## Сравнение подходов

| Подход | Механизм | Преимущества | Недостатки |
|---|---|---|---|
| wait/notify | `synchronized` + `wait()` / `notifyAll()` | Простой и понятный; корректное завершение через `closeShop()` | Требует аккуратной работы с монитором; `while`-условие обязательно |
| Semaphore | `Semaphore` + `tryAcquire` / `release` | Явное разделение сигналов; неблокирующий отказ (`tryAcquire`) | Больше объектов синхронизации; сложнее корректно завершить |
