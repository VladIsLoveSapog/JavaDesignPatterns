# Создание потоков

В Java есть два основных способа создания потоков, которые реализуются в четырёх вариантах:

1. Наследование от класса `Thread` — вариант через extends.
2. Реализация интерфейса `Runnable` — три варианта: отдельный класс, анонимный класс, лямбда.

# Класс Thread

`Thread` — это класс из пакета `java.lang`, представляющий поток выполнения в Java. Когда вызывается метод `start()`,
JVM инициирует создание нового **нативного потока операционной системы** через системные вызовы. Именно в этом
OS-потоке будет выполнен метод `run()`.

## Жизненный цикл потока

| Состояние       | Описание                                                                 |
|-----------------|--------------------------------------------------------------------------|
| `NEW`           | Поток создан (`new Thread(...)`), но `start()` ещё не вызван.            |
| `RUNNABLE`      | Поток выполняется или готов к выполнению (ожидает процессор от ОС).      |
| `BLOCKED`       | Поток ожидает захвата монитора (`synchronized`-блок или метод).          |
| `WAITING`       | Поток ждёт бесконечно: `wait()`, `join()` без таймаута, `park()`.        |
| `TIMED_WAITING` | Поток ждёт с таймаутом: `sleep(ms)`, `wait(ms)`, `join(ms)`.             |
| `TERMINATED`    | Поток завершил выполнение `run()` (штатно или через исключение).         |

## Ключевые методы класса Thread

| Метод                          | Описание                                                                                   |
|--------------------------------|--------------------------------------------------------------------------------------------|
| `start()`                      | Запускает поток — создаёт нативный поток ОС и вызывает `run()` в нём.                     |
| `run()`                        | Тело потока. Вызывать напрямую **нельзя** — выполнится в текущем потоке, без нового.       |
| `sleep(ms)` *(static)*         | Приостанавливает **текущий** поток на указанное время. Бросает `InterruptedException`.     |
| `interrupt()`                  | Устанавливает флаг прерывания; выбрасывает `InterruptedException` в блокирующих методах.  |
| `join()`                       | Текущий поток блокируется до завершения **этого** потока.                                  |
| `yield()` *(static)*           | Подсказка планировщику ОС уступить процессор другому потоку.                               |
| `currentThread()` *(static)*   | Возвращает ссылку на поток, в котором вызван.                                              |
| `getName()` / `setName()`      | Имя потока (отображается в логах и отладчике).                                             |
| `setPriority(int)`             | Приоритет от `MIN_PRIORITY` (1) до `MAX_PRIORITY` (10); подсказка для планировщика ОС.    |
| `setDaemon(boolean)`           | Помечает поток как фоновый (daemon). Daemon-потоки завершаются вместе с последним user-потоком. |
| `getState()`                   | Возвращает текущее состояние потока (`Thread.State`).                                      |
| `getId()` / `threadId()`       | Уникальный числовой идентификатор потока.                                                  |

# Интерфейс Runnable

`Runnable` — это интерфейс из пакета `java.lang` с единственным абстрактным методом `run()`. Аннотация
`@FunctionalInterface` означает, что его можно реализовать лямбда-выражением.

**Главная идея: задача ≠ поток.**
`Runnable` описывает *что делать*, а `Thread` — *как и когда запустить*. Один экземпляр `Runnable` можно передать
сразу в несколько потоков `Thread`.

**Почему `Runnable` предпочтительнее `extends Thread`:**
- В Java нет множественного наследования классов — если класс уже наследуется от чего-то, `extends Thread` невозможен.
- Реализация `Runnable` не смешивает логику задачи с механизмом управления потоком.
- `Runnable` — `@FunctionalInterface`, то есть заменяется лямбдой без создания отдельного класса.

**Ограничение `Runnable`:** метод `run()` не возвращает значение и не может пробрасывать checked-исключения
(только `RuntimeException`). Для задач с возвратом результата используется `Callable<V>` — см. раздел
«Дополнительные способы» в конце.

## Внимание!

Переопределяется метод `run()`, а запускается поток через `start()`. При прямом вызове `run()` код выполнится
в **текущем** потоке (обычно `main`) — нового потока создано не будет.

# Соответсвие Java поток и потоков ОС

В современных реализациях Java Virtual Machine (JVM) Java-потоки обычно реализуются с использованием нативных потоков
операционной системы (ОС).

Когда вы создаете и запускаете новый поток в Java с помощью класса ```Thread``` или через ```ExecutorService```, JVM
создает соответствующий нативный поток ОС. Это означает, что каждый Java-поток сопоставлен один-к-одному с потоком
операционной системы. Такая модель позволяет JVM использовать возможности многозадачности и многопоточности,
предоставляемые ОС, включая планирование потоков, управление приоритетами и распределение на разные ядра процессора.

## Подробности реализации

- **Создание потока**: Вызов метода ```start()``` на объекте ```Thread``` инициирует создание нового нативного потока ОС
  через системные вызовы.

```
Thread thread = new Thread(() -> {
    // Код, выполняемый в новом потоке
});
thread.start(); // Создает нативный поток ОС
```

- **Планирование и управление**: Операционная система отвечает за планирование выполнения потоков, распределяя их по
  доступным процессорам и ядрам.

# Прерывание потоков

В Java безопасное и контролируемое завершение потока может быть достигнуто с помощью механизма
прерываний (```interruptions```). **Прерывание потока** — это способ сообщить потоку, что ему следует прекратить свою
работу. Это особенно важно, когда поток выполняет длительную или бесконечную задачу, и необходимо иметь возможность
корректно остановить его выполнение.

**Прерывание потока** — это сигнал, который отправляется потоку, чтобы указать, что ему следует прекратить выполнение.
Прерывание не заставляет поток немедленно остановиться; вместо этого поток должен сам проверить, был ли он прерван, и
корректно завершить свою работу.

## Как работает прерывание потока?

1. **Отправка сигнала прерывания**: Используется метод ```interrupt()``` у объекта ```Thread```.

```
thread.interrupt();
```

2. **Обработка прерывания в потоке**:
    - Поток должен периодически проверять, был ли он прерван, с помощью методов ```Thread.interrupted() ```
      или ```isInterrupted()```.
    - Если поток находится в состоянии ожидания (например, вызовы ```sleep()```, ```wait()```, ```join()```), то при
      прерывании будет выброшено ```InterruptedException```.

# Дополнительные способы работы с потоками

Следующие механизмы выходят за рамки базового `Thread`/`Runnable` и не рассматриваются в данном курсе:

- **`Callable<V>` + `Future<V>`** — задача с возвратом результата и возможностью бросать checked-исключения.
- **`ExecutorService` / `Executors`** — управление пулом потоков; не создавать потоки вручную на каждую задачу.
- **`CompletableFuture`** — асинхронные цепочки операций (thenApply, thenCompose, exceptionally и др.).
- **`ForkJoinPool` / `RecursiveTask`** — divide and conquer: рекурсивное разбиение задачи на подзадачи.
- **`ScheduledExecutorService`** — отложенный или периодический запуск задач.
- **`ThreadLocal<T>`** — переменные уровня потока: каждый поток имеет свою независимую копию.
- **Virtual Threads (Java 21+)** — лёгкие виртуальные потоки, управляемые JVM, а не ОС; масштабируются до миллионов.
