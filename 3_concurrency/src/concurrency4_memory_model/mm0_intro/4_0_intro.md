# Модель памяти Java (JMM)

**Java Memory Model (JMM)** — это спецификация, которая определяет, как потоки взаимодействуют через общую память: когда
изменения, сделанные одним потоком, становятся видны другим потокам, и в каком порядке операции с памятью могут
наблюдаться.

## Проблемы многопоточного доступа к памяти

В многопоточной программе каждый поток может иметь собственную **рабочую копию** переменных в регистрах или кеше
процессора. Из-за этого возникают три класса проблем:

### 1. Проблема видимости (Visibility)

Поток A записал новое значение в переменную, но поток B по-прежнему читает устаревшее значение из своего кеша.

```
Поток A:  flag = true;       // записал в кеш A
Поток B:  while (!flag) {}  // читает свой кеш — может зависнуть навсегда
```

### 2. Проблема переупорядочивания (Reordering)

Компилятор и процессор вправе переупорядочивать инструкции для оптимизации, если это не нарушает однопоточную
семантику. В многопоточной среде это может привести к неожиданному поведению.

```
// Исходный код:
data = 42;
ready = true;

// После оптимизации компилятором может стать:
ready = true;     // другой поток увидит ready=true раньше, чем data=42
data = 42;
```

### 3. Проблема атомарности (Atomicity)

Некоторые операции, выглядящие как одна инструкция, на уровне процессора состоят из нескольких шагов.
Например, `counter++` — это три операции: прочитать, увеличить, записать. Между ними может вмешаться другой поток.

## Инструменты JMM

| Инструмент | Решает | Не решает |
|------------|--------|-----------|
| `volatile` | Видимость + переупорядочивание | Атомарность составных операций |
| `Atomic`-классы | Видимость + атомарность (lock-free) | Атомарность операций над несколькими переменными |
| `synchronized` / `Lock` | Всё перечисленное, но с блокировкой | — |

## Happens-Before

JMM вводит понятие **happens-before** — гарантия порядка видимости операций:
- Если операция A _happens-before_ B, то все изменения, сделанные A, видны B.
- Запись в `volatile` переменную _happens-before_ любого последующего чтения этой переменной.
- `unlock()` монитора _happens-before_ любого последующего `lock()` того же монитора.

## Содержание раздела

- **`mm1_volatile`** — ключевое слово `volatile`: видимость и запрет переупорядочивания.
- **`mm2_atomic`** — классы `java.util.concurrent.atomic`: атомарные операции без блокировок (CAS).
